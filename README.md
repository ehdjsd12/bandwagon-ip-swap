## 第一步：先确认你的 IP 真的被封了

IP 被封的症状和 VPS 死机、网络临时抖动很像，不要一上来就花钱换 IP，先确认。

方法很简单：登录搬瓦工后台，进入对应 VPS 的 KiwiVM 控制面板，在浏览器地址栏把 URL 末尾的 `main.php` 替换成 `main-exec.php?mode=blacklistcheck`，点击 Test Main IP。

结果判断：

- 显示 **IP BLOCKED** → 确认被封，需要换 IP
- 显示 **IP NOT BLOCKED** → IP 没问题，检查一下 SSH 端口或者本地网络配置

另一个判断方式：国外网络能访问你的 VPS，国内 TCP 全线超时。这种"墙内不通、墙外正常"的表现，基本就是 IP 被国内封了。

## 搬瓦工换了 IP 后怎么样？换完要做这几件事

换完 IP 之后，大部分情况下可以直接用新 IP 连接，但有几个地方需要手动更新一下，不然还是用不了。

### 1. 更新 SSH 连接配置

你的 SSH 客户端（Xshell、Termius、PuTTY 等）里保存的主机地址还是旧 IP，需要改成新 IP，重新连接。

### 2. 更新域名 DNS 解析

如果你在这台 VPS 上跑着网站，域名原来解析到旧 IP，现在旧 IP 没了，网站自然访问不了。需要去你的域名注册商或 DNS 服务商那边，把 A 记录改成新 IP。改完之后等 DNS 生效，通常十几分钟到几小时不等，取决于 TTL 设置。

### 3. 检查应用里写死的 IP 配置

有些程序会在配置文件里直接写 IP 地址。换完 IP 后，如果你的服务还是连不上或者转发失效，大概率是这里的问题，挨个检查一遍配置文件，把旧 IP 换掉。

### 4. 验证新 IP 好不好用

换完 IP 之后，官方保证新 IP 全球可达。但也可以自己验一下：用 ping.pe 测测新 IP 国内节点是否正常，确认没问题之后再继续配置。

换完之后一切正常的话，就是这样了。VPS 上跑着的服务不需要重新搭建，数据一条没丢。

## 常见问题（FAQ）

**搬瓦工换了 IP 后数据还在吗？**

在的。换 IP 只是给你的 VPS 重新分配一个 IPv4 地址，服务器上所有数据、文件、已安装的程序全部原样保留，不需要重新配置系统。

**搬瓦工换 IP 要多久？**

付款之后，快的话几分钟就完成，一般半小时内搞定。极少数情况需要客服手动处理，最长 24 小时内完成。

**搬瓦工换 IP 可以免费吗？**

不行。搬瓦工之前有过免费换 IP 的服务，每隔固定周期可以免费换一次，但这个功能在 2019 年就已经永久下线了。现在只有付费这一条路，$8.79/次。

**搬瓦工 IP 被封了还能迁机房吗？**

不能。IP 被封的状态下，KiwiVM 后台的机房迁移功能会被禁用，直接提示不可用。必须先付费换 IP，换完之后机房迁移功能才会恢复正常。

**换完 IP 之后网站还是访问不了怎么办？**

多半是 DNS 解析还没更新。换 IP 后要手动把域名 A 记录改成新 IP，然后等 DNS 在全球生效，一般需要几分钟到几小时。如果 DNS 已经更新了还是不行，检查一下防火墙规则和应用配置文件里有没有写死的旧 IP。
